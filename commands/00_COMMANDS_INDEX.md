## The docs

- [the-prompt-pack-pattern](./the-prompt-pack-pattern.md) - how to build a FULL prompt pack per orchestrator: the resume/boot block plus the role's complete situational set, generated by the role itself in the sandbox and re-run validated, with the hard rules baked into every fenced block so a pasting session cannot drift.
- [example-orchestrator-pack](./example-orchestrator-pack.md) - a generalized FULL pack for a generic orchestrator: a resume/boot block plus situational blocks (fix-it, wave-run, stage-it, ship-it, incident, retro, builder-dispatch, builder-boot), each a fenced block with the gates baked in (stage-not-deploy, verify-independently). Fully generic placeholders.

## Conventions

- **Every orchestrator gets a pack** in its own `commands/` folder. When a new orchestrator is minted, its pack lands there and a row is added here.
- **New prompt?** Add a labeled fenced block to the owning role's pack (UPDATE, do not duplicate). Bake the role's hard rules into the block. No em dashes.
- **Keep it current.** When a role's resume or ceremony changes, regenerate the affected blocks and re-run validate them, and reconcile this index in the same change.
